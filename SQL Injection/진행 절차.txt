01. SQL Injection
  - SQL Injection은 악의적인 SQL 코드를 삽입하여 데이터베이스에 대한
    비인가된 액세스를 얻는 공격입니다.

02. SQL Injection 작동 원리
  01) 악의적인 사용자는 애플리케이션의 입력 필드나 매개변수에 SQL 쿼리의
       일부 또는 완전한 SQL 문을 삽입합니다.
  02) 삽입된 SQL 코드는 애플리케이션 서버로 전송되고, 데이터베이스에 실행
       될 때 악의적인 명령이 함께 실행될 수 있습니다.
  03) 코드가 실행되면 공격자는 데이터베이스를 조작하거나 민감한 정보를
       액세스 합니다.

03. SQL Injection 방지를 위한 조치
  - 매개변수화된 쿼리 사용: SQL 쿼리에서 사용자의 입력 값을 직접 삽입하는
    대신 매개변수화된 쿼리를 사용하여 안전하게 처리합니다.
  - 사용자 입력 값을 이스케이핑하여 SQL 구문에 영향을 주지 않도록 하여
    악의적인 코드가 입력 값으로 해석되지 않도록 보호합니다.
